# 智慧健康日誌風險評估系統

## 一、選擇題目

本專案選擇題目 A：智慧健康日誌風險評估系統。

使用者可以輸入每日健康資料，包含日期、睡眠時數、步數與心情分數。系統會透過決策樹判斷目前健康風險等級，並將結果儲存在 SQLite 資料庫中。

---

## 二、系統功能

本系統包含以下功能：

1. 新增健康日誌
2. 顯示所有健康日誌紀錄
3. 修改指定健康日誌
4. 刪除指定健康日誌
5. 使用決策樹自動判斷風險等級
6. 以顏色顯示風險結果：
   - 低風險：綠色
   - 中風險：黃色
   - 高風險：紅色

---

## 三、使用技術

- Frontend：HTML、CSS、JavaScript
- Backend：Node.js、Express
- Database：SQLite
- API：RESTful API
- DM/ML 方法：決策樹分類

---

## 四、資料表設計

資料表名稱：health_logs

| 欄位名稱 | 型態 | 說明 |
|---|---|---|
| id | INTEGER | 主鍵，自動增加 |
| log_date | DATE | 健康日誌日期 |
| sleep_hours | REAL | 睡眠時數 |
| steps | INTEGER | 當日步數 |
| mood_score | INTEGER | 心情分數，1～10 |
| risk_level | TEXT | 決策樹判斷後的風險等級 |

---

## 五、API 端點

| 方法 | 路徑 | 功能 |
|---|---|---|
| GET | /health-logs | 取得所有健康日誌 |
| POST | /health-logs | 新增健康日誌 |
| PUT | /health-logs/:id | 修改指定健康日誌 |
| DELETE | /health-logs/:id | 刪除指定健康日誌 |
| GET | /health-logs/risk | 測試決策樹風險判斷 |

---

## 六、決策樹邏輯說明

本系統使用睡眠時數、步數與心情分數作為決策樹的判斷特徵。

判斷邏輯如下：

1. 如果睡眠時數少於 5.5 小時：
   - 若步數少於 4000，判斷為高風險
   - 若步數不少，但心情分數小於等於 4，判斷為高風險
   - 其他情況判斷為中風險

2. 如果睡眠時數大於等於 7 小時：
   - 若步數大於等於 6000 且心情分數大於等於 6，判斷為低風險
   - 否則判斷為中風險

3. 如果睡眠時數介於 5.5 到 7 小時之間：
   - 若心情分數小於等於 4，判斷為中風險
   - 其他情況判斷為低風險

## 七、種子資料

本專案使用 seed.js 產生 90 筆健康日誌資料，包含：

- 30 筆高風險資料
- 30 筆中風險資料
- 30 筆低風險資料

資料會寫入 health.db 中的 health_logs 資料表。

---

## 八、執行方式

安裝套件：

npm install

產生種子資料：

node seed.js

啟動伺服器：

node server.js

開啟瀏覽器：

http://localhost:3000

---

## 九、資料流說明

本系統的資料流如下：

前端表單輸入資料  
→ JavaScript 使用 fetch 呼叫 API  
→ Express 後端接收資料  
→ 後端執行決策樹判斷風險等級  
→ 將資料與風險結果寫入 SQLite  
→ 前端重新取得資料並顯示在表格中

---

## 十、專案總結

本專案完成了一個結合前端、後端、資料庫與決策樹分類的小型健康日誌系統。透過此專案，可以理解資料如何從前端流向 API，再進入資料庫，並透過簡單的決策樹模型產生分類結果。
